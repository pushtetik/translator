# Транслятор (интерпретатор) подмножества BASIC

Транслятор для ограниченного подмножества языка программирования **BASIC**, реализованный на C++

---

## Возможности

- **Лексический анализ** — конечный автомат, распознающий токены входной программы
- **Синтаксический анализ** — метод рекурсивного спуска, построение синтаксического дерева
- **Семантический анализ** — проверка объявления переменных, обнаружение неиспользуемых переменных
- **Интерпретация** — рекурсивный обход синтаксического дерева и непосредственное выполнение программы
- **Диагностика ошибок** — подробные сообщения для лексических, синтаксических и семантических ошибок

## Поток данных между модулями

| Модуль | Вход | Выход | Назначение |
|--------|------|-------|------------|
| **Лексический анализатор** (сканер) | Исходный код (строка), таблица ключевых слов, разделителей | Последовательность токенов + таблицы (констант, идентификаторов, ошибок) | Преобразует текст программы в поток лексем, отбрасывает пробелы и комментарии |
| **Синтаксический и семантический анализатор** (парсер) | Грамматика языка, последовательность токенов, таблица идентификаторов | Абстрактное синтаксическое дерево (AST) + аннотированная таблица символов | Проверяет соответствие грамматике, строит внутреннее представление программы и выполняет смысловой контроль (объявление переменных, совместимость типов) |
| **Интерпретатор** | Абстрактное синтаксическое дерево (AST) | Результат выполнения программы, текущее значение переменных | Обходит дерево и выполняет узлы в соответствии с их семантикой |

## Поддерживаемый синтаксис

### Объявление переменных
```basic
DIM X, Y AS INTEGER
DIM RESULT AS INTEGER
```

### Присваивание и арифметика
```basic
X = 5
Y = X * 2 + 3
RESULT = X ^ 2 MOD 7
```

Поддерживаемые операции:

| Оператор | Описание              | Пример |
|----------|-----------------------|--------|
| `+`      | Сложение              | `X + 5` |
| `-`      | Вычитание             | `X - 3` |
| `*`      | Умножение             | `A * B` |
| `/`      | Целочисленное деление | `10 / 3` → `3` |
| `^`      | Возведение в степень  | `2 ^ 3` → `8` |
| `MOD`    | Остаток от деления    | `10 MOD 3` → `1` |
| `(` `)`  | Изменение приоритета  | `(A + B) * C` |

### Приоритет операторов (от высшего к низшему)

1. **Скобки** `(` `)` — изменение приоритета вычислений
2. **Унарные** `+` и `-`
3. **Возведение в степень** `^` (правоассоциативный)
4. **Умножение** `*`, **деление** `/`, **остаток** `MOD`
5. **Сложение** `+`, **вычитание** `-`
6. **Операции сравнения** `=`, `<`, `>`, `<=`, `>=`, `<>`
7. **Логическое** `AND`
8. **Логическое** `OR`

### Ввод / вывод
```basic
INPUT X
PRINT "Результат: ", X + Y
PRINT X * 2
```

### Условный оператор — однострочная форма
```basic
IF X > 0 THEN PRINT "Положительное" ELSE PRINT "Не положительное"
```

### Условный оператор — многострочная форма
```basic
IF X > 100 THEN
    PRINT "Больше ста"
ELSEIF X > 50 THEN
    PRINT "Больше пятидесяти"
ELSEIF X > 0 THEN
    PRINT "Положительное"
ELSE
    PRINT "Ноль или меньше"
END IF
```

## ▶️ Запуск

1. Создайте файл `program.txt` в директории с исполняемым файлом
2. Запишите программу на подмножестве BASIC
3. Запустите транслятор

```bash
./translator
```

После выполнения в директории появятся:
- `tokens.txt` — последовательность токенов
- `syntax_tree.txt` — синтаксическое дерево программы

## Пример программы

```basic
DIM A, B, SUM, PROD AS INTEGER

INPUT A
INPUT B

SUM = A + B
PROD = A * B

PRINT "Сумма: ", SUM
PRINT "Произведение: ", PROD

IF SUM > PROD THEN
    PRINT "Сумма больше"
ELSE
    PRINT "Произведение больше или равно"
END IF
```

### Составные операторы сравнения

Некоторые операторы сравнения образуются комбинацией базовых разделителей:

| Оператор | Состав | Описание           |
|----------|--------|--------------------|
| `<=`     | `<` + `=` | Меньше или равно |
| `>=`     | `>` + `=` | Больше или равно |
| `<>`     | `<` + `>` | Не равно         |

## Диагностика ошибок

Транслятор обеспечивает многоуровневую диагностику ошибок с указанием места возникновения и понятным описанием проблемы.

### Уровни диагностики

| Уровень | Где обнаруживается | Что фиксируется |
|---------|---------------------|-----------------|
| **Лексические ошибки** | Лексический анализатор | Недопустимые символы, нераспознанные лексемы |
| **Синтаксические ошибки** | Синтаксический анализатор | Нарушение грамматики, несоответствие ожидаемой структуре |
| **Семантические ошибки** | Семантический анализатор | Нарушение правил использования языка |
| **Ошибки выполнения** | Интерпретатор | Ошибки, возникающие в процессе исполнения (деление на ноль и т.п.) |
| **Предупреждения** | Семантический анализатор | Нефатальные нарушения (неиспользуемые переменные) |

### Классификация ошибок

#### 1. Лексические ошибки
Возникают, когда во входном потоке встречается символ, не принадлежащий алфавиту языка.  
**Диагностика:** таблица ошибок с указанием строки, позиции и недопустимого символа.

#### 2. Синтаксические ошибки
Возникают, когда последовательность токенов не соответствует грамматике языка.  
**Диагностика:** сообщение с указанием ожидаемой конструкции и фактически найденной.

#### 3. Семантические ошибки
Возникают при нарушении правил использования языка:
- Обращение к необъявленной переменной
- Повторное объявление переменной
**Диагностика:** сообщение с указанием имени переменной и характера нарушения.

#### 4. Ошибки выполнения
Возникают в процессе интерпретации программы:
- Арифметические ошибки (деление на ноль, недопустимые операции)
- Ошибки ввода-вывода
**Диагностика:** сообщение с описанием ошибки, после чего выполнение программы прерывается.

#### 5. Предупреждения
Не прерывают выполнение, но информируют о потенциальных проблемах:
- Переменная объявлена, но не используется
**Диагностика:** сообщение с указанием имени переменной, программа продолжает работу.

### Примеры диагностических сообщений

| Тип ошибки | Сообщение |
|------------|-----------|
| Лексическая | `Ошибка лексического анализа: недопустимый символ '?' в строке 3, позиции 12` |
| Синтаксическая | `Синтаксическая ошибка: ожидалось 'THEN', но найдено 'PRINT' в строке 5` |
| Синтаксическая | `Синтаксическая ошибка: ожидалось число, идентификатор или '('` |
| Семантическая | `Семантическая ошибка: переменная 'X' не объявлена` |
| Семантическая | `Семантическая ошибка: переменная 'Z' объявлена дважды` |
| Ошибка выполнения | `Ошибка выполнения: деление на ноль` |
| Предупреждение | `Предупреждение: переменная 'UNUSED' объявлена, но не используется` |

### Обработка ошибок ввода пользователя (INPUT)

Интерпретатор устойчив к некорректному вводу данных. При вводе нечислового значения:
1. Очищается флаг ошибки потока ввода (`cin.clear()`)
2. Сбрасывается буфер ввода (`cin.ignore()`)
3. Выводится повторный запрос
4. Цикл продолжается до получения корректного целого числа

## Правила языка

- Все переменные объявляются через `DIM ... AS INTEGER` до использования
- Имена переменных: латинские буквы, цифры, `_`; начинаются с буквы
- Ключевые слова регистронезависимы (`if` = `IF`)
- Строки в `PRINT` заключаются в двойные кавычки
- Отрицательные числа — через унарный минус: `X = -5`
- Перенос строки является разделителем операторов в многострочных конструкциях
- Переменные, объявленные через `DIM`, автоматически инициализируются нулём
